vulncms靶机记录

arp-scan -l 扫描靶机地址

一.信息收集

nmap：收集IP地址下端口情况

nmap -T4 -sV -p- -A 192.168.219.132

发现22，80，5000，8081，8888，8889，9001端口都是开启的

（一）80端口 : web服务端口

1.目录扫描

dirsearch -u 192.168.219.132：80

发现一些可以访问的目录地址：/about.html, /home.html, /robots.txt

（二）5000端口：web服务

80端口和5000端口是两种不同的web服务端口，80端口一般是默认web服务访问端口，5000则可能是用于其他用途的自定义端口

1.目录扫描

扫描出很多目录但大部分打不开，发现是wordpress框架，使用wpscan扫描

发现版本，但未扫描出漏洞

（三）8081端口

1.目录扫描

访问http://192.168.219.132:8081/ 我们使用wrap插件发现有joomla系统，使用joomscan进行扫描

发现版本信息和漏洞

在msf中搜索该漏洞看看用没有利用

（四)9001端口

1.目录扫描

发现Drupal 7.54

在msfconsole中搜索

提权利用。

尝试提权。

1.可以利用提权工具去探测可能存下提权的地方。

2.手动搜索，自己找账号密码

cd /var/

cd www

cd joomla

cat configuration.php

发现joomla登陆账号密码

同理，

cd wordpress

cat wp-config.php

我们使用elliot身份登陆

ssh elloit@192.168.219.132

输入密码

连接成功！

看看该身份下有没有权限，发现没有权限

使用python pty模块新建终端即可绕过

打开了tyrell用户

ssh连接用户

sudo -l

发现有一个文件路径是root身份执行

GTFOBins

网站上搜索journalctl

得到使用方法

输入，获得权限

linux提权

在黑客的攻击渗透中，会想尽办法通过「某个漏洞」获取到目标主机系统上的一个 shell 用于执行系统命令，从而达到控制目标主机的目的。而当他获取到的初始 shell 为低权限用户时，需要想办法提升权限为高权限用户，*这个过程就叫做系统权限提升，简称「提权」 。

一、passwd文件提权

/etc/passwd和/etc/shadow 文件是linux系统登录认证的关键文件。

提权就是利用/etc/passwd 文件的可写入权限，我们可以写入一个其他用户进去。

如果系统运维人员对passwd或者shadow文件的内容或权配置有误，则可以被利用进行系统提权。

方法一：如果具有 /etc/passwd 的w(写入权限)，可以直接添加一个root权限的用户。

方法二：如果/etc/passwd 中存储root用户密码哈希，可以用john进行破解。

二、具体操作——linux提权，利用/etc/passwd 文件

1.首先，查看/etc/passwd的权限。

ls -lh /etc/passwd 查看文件的具体信息，其中包括不同的用户对该文件的权限

r（读） w（写） x（执行）

2.记录行的基本属性

linux系统中的用户在/etc/passwd 文件中有一个对应的记录行，它记录了此时这个用户的一些基本属性。

其包含的信息顺序如下：

用户名 加密密码 用户ID 组ID 用户名 用户家目录 登录shell

3.构造用户，密码加盐
（加盐，就是人为的通过一组随机字符与用户原密码的组合形成一个新的字符，从而增加破译的难度。）

首先我们需要构建一个用户，在密码占位符处指定密码，并且将UID设置为0（管理员的UID为0，用户UID为0的都可以作为管理员），并将其添加到/etc/passwd 文件中。

首先使用 perl语言 生成带有盐值的密码： perl -le ‘print crypt(“test”,”addedsalt”)’

echo “test:adMpHktIn0tR2:0:0:User_like_root:/root:/bin/bash” >>/etc/passwd 执行这条命令将test用户的信息加入到/etc/passwd文件里面；

再 ifconfig 查看本地ip地址 eth0

使用用户名密码登录：ssh test@本地ip地址 （登录主机），查看权限。

在渗透测试中，高权限 shell 能带来以下好处：
  • 高权限能对更多的文件进行「增删改查」操作，便于进一步收集主机系统中的敏感信息
  • Linux 系统的 root 权限可获取 shadow 文件中的密码 Hash，若内网环境中存在「账户/密码复用」的情况，可用于横向扩展
  • Windows 系统中的 system 权限可提取内存中的密码 Hash，可进一步用于域渗透

三，暴力提权

1.通过su命令爆破root密码

su命令爆破使用的工具是sucrack

su root 切换为root用户

2.通过ssh爆破密码

SSH 服务的配置文件有为两个，分别是：

  • /etc/ssh/ssh_config : 客户端的配置文件
  • /etc/ssh/sshd_config : 服务端的配置文件

当且仅当/etc/ssh/sshd_config 中 PermitRootLogin 设置为 yes，root 用户才能登录 ssh：

再通过ssh爆破root用户密码之前，我们需要先使用 cat /etc/ssh/sshd_config |grep -i permitrootlogin 来查询靶机是否允许root用户通过ssh登录。

当 PermitRootLogin 被设置为 yes 时，才用尝试 SSH 暴力破解，否则就没必要在尝试了。

注意，此时我们已经获取到目标靶机的初始 shell，因此可以直接查看靶机上的 sshd 配置文件，后续的 ssh 爆破也是在本地进行的。

ssh协议爆破使用到的工具是hydra。

参数解释：

  • -l : 指定用户名
  • -P : 指定爆破字典
  • -t : 指定爆破线程
  • -V : 查看破解详情、进度
